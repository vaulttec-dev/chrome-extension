# Запис Meet → Drive

Мінімалістичне Chrome-розширення (Manifest V3): під час дзвінка в Google Meet з'являється
кнопка **● Запис**. Один клік → просте «Поділитися цією вкладкою?» → йде запис **відео +
звуку зустрічі + вашого мікрофона**. Коли ви виходите з дзвінка — запис **зупиняється сам**
і файл зберігається.

За замовчуванням файл падає у теку **«Завантаження»**. Якщо налаштувати Google Drive
(розділ нижче) — вивантажується одразу в Drive, з групуванням по теках:
`Запис зустрічей / РРРР-ММ-ДД / Meet <код> <дата> <час>` — кожна зустріч у власній теці
(відео + конспект разом), а ті — у теці дня.

Якщо додати **Gemini API-ключ** (розділ «Авто-конспект» нижче) — одразу після запису
розширення робить **структурований конспект українською** і кладе його окремим
**Google Документом** у ту саму теку.

Працює на безкоштовному `@gmail.com`. Записує навіть чужі дзвінки, де ви не організатор.

---

## Як це влаштовано

Захоплення, запис **і великі аплоади** (Drive / заливання у Gemini) відбуваються **в самій
сторінці Meet** (content script) — бо там уже є дозвіл на мікрофон, а головне: відео-файл не
ганяється через `sendMessage`, а вантажиться `fetch`'ем прямо звідси. Service worker дає
content script'у OAuth-токен (бо `chrome.identity` доступний лише там) і веде дрібну фонову
обробку Gemini через `chrome.alarms` — незалежно від того, чи відкрита вкладка Meet.

| Файл | Роль |
|------|------|
| `manifest.json` | Дозволи (`identity`, `storage`, `downloads`, `alarms`), OAuth, реєстрація скриптів. |
| `content.js` / `content.css` | Кнопка в Meet, детекція дзвінка, захоплення (`getDisplayMedia` + мікрофон), `MediaRecorder`, аплоад відео на Drive (або локально) і заливання відео в Gemini. |
| `gdrive.js` / `gemini.js` | Чисті функції Drive / Gemini (приймають токен/ключ) — спільні для content script і service worker, без дублювання. |
| `background.js` | OAuth-токен для content script, резервна зупинка, фоновий конспект через `chrome.alarms` (Files API → `generateContent` → Google Doc / `.txt`). |
| `popup.html` / `popup.js` | Статус запису + резервна кнопка «Зупинити». |
| `icons/` | Іконки (червона крапка). |

Потік: клік «● Запис» → `getDisplayMedia({preferCurrentTab})` + `getUserMedia` (мікрофон) →
мікс аудіо → `MediaRecorder` (WebM) → на стоп `content.js` бере токен у `background.js` і
вантажить відео на Drive (інакше локально), потім (якщо є ключ) заливає відео в Gemini й
передає `background.js` дрібну задачу — той по `chrome.alarms` чекає обробки, генерує конспект
і кладе Google Документом у ту саму теку.

## Використання
1. Зайдіть у дзвінок Google Meet.
2. Натисніть **● Запис** (внизу екрана) → у діалозі **«Поділитися»** (галочка звуку вкладки за потреби).
3. На іконці розширення — бейдж **REC**.
4. Завершіть дзвінок — запис зупиниться сам і збережеться.
   (Також можна натиснути **■ Зупинити запис** на кнопці або в попапі.)
5. Файл `Meet <код> <дата>.webm` — у теці «Завантаження» (або в Drive, якщо налаштовано).

## (Опційно) Збереження одразу в Google Drive

Без цього файли зберігаються локально. Щоб писати прямо в Drive:

1. `chrome://extensions` → увімкніть **Режим розробника** → **Завантажити розпаковане** → ця тека. Скопіюйте **ID** розширення.
2. [console.cloud.google.com](https://console.cloud.google.com) → створіть проєкт.
3. **APIs & Services → Library** → увімкніть **Google Drive API**.
4. **OAuth consent screen** → *External* → назва, email; scope `.../auth/drive.file`; додайте себе в *Test users*.
5. **Credentials → OAuth client ID** → тип **Chrome Extension** → вставте **ID розширення** → скопіюйте **Client ID**.
6. У `manifest.json` замініть `ВАШ_CLIENT_ID.apps.googleusercontent.com` на свій Client ID → **↻** на картці розширення.

Після цього запис автоматично йтиме в Drive (папка «Запис зустрічей»), а якщо вивантаження
не вдасться — все одно збережеться локально.

## (Опційно) Авто-конспект через Gemini

Якщо вписати Gemini API-ключ — після кожного запису розширення надсилає **відео** в Gemini
і отримує **конспект українською** (короткий підсумок, ключові тези, ухвалені рішення,
завдання/дедлайни, відкриті питання). Конспект зберігається **Google Документом**
«Meet … — конспект» у теці тієї ж зустрічі (поряд із відео; або локально `.txt`, якщо Drive
недоступний).

**Налаштування:**
1. [aistudio.google.com/apikey](https://aistudio.google.com/apikey) → **Create API key**
   (можна в тому ж проєкті, що й для Drive).
2. Відкрийте popup розширення → вставте ключ у поле **«Gemini API-ключ»** → **«Зберегти ключ»**.

**Приватність (важливо):** на **безкоштовному** тарифі Gemini Google може використовувати
надіслані аудіо/відео та відповіді для покращення своїх моделей. Для запису зустрічей, де
можуть бути інші люди, це варто враховувати. Щоб цього уникнути — створіть ключ у проєкті з
**увімкненим білінгом (платний тариф)** або використовуйте Vertex AI: на них дані для
навчання **не використовуються**. Код розширення при цьому той самий — різниця лише в тарифі
проєкту, у якому видано ключ.

Технічна примітка: надсилається саме відео (`video/webm`) з `mediaResolution: LOW` — бо
`MediaRecorder` дає `webm/opus`, який Gemini не приймає як чисте аудіо. Відео заливається в
Gemini Files API прямо з content script (де воно вже в пам'яті), а очікування обробки й
генерацію конспекту веде service worker фоново через `chrome.alarms`, тож вкладку Meet можна
закрити одразу. Низька роздільність кадрів тримає запит у межах лімітів навіть для довгих
зустрічей.

## Обмеження
- **Старт лише за кліком** (Chrome не дозволяє почати захоплення без жесту користувача).
- **Діалог «Поділитися»** показується щоразу — це вимога Chrome для запису екрана/вкладки.
- **Детекція кінця дзвінка** спирається на URL/DOM Meet; за зміни верстки Google може
  знадобитися правка в `content.js` (на цей випадок є ручна зупинка).
- **Дуже довгі зустрічі**: файл тримається в пам'яті до завершення; для багатогодинних
  записів згодом варто додати запис шматками.
- **Звук зустрічі** пишеться лише якщо у вкладці реально щось звучить (інші учасники) і
  поставлено галочку «звук вкладки»; ваш голос іде з мікрофона.
